# 异构 NMS 框架扩展：改动总结与使用说明

本文档对应 `scratch/heterogeneous-nms/framework/heterogeneous-nms-framework.cc` 与相关脚本的扩展功能说明。

---

## 运行指令（复制即用）

在项目根目录 `ns-3-dev` 下执行。

### 编译

```bash
./ns3 build heterogeneous-nms-main
```

### 运行仿真

```bash
# 默认：仿真 100s，MESH 拓扑，单通道
./ns3 run heterogeneous-nms-main

# 指定仿真时长（秒）
./ns3 run heterogeneous-nms-main -- --simTime=50

# Ad-hoc 拓扑：0=MESH, 1=STAR, 2=CHAIN
./ns3 run heterogeneous-nms-main -- --topology=0
./ns3 run heterogeneous-nms-main -- --topology=1
./ns3 run heterogeneous-nms-main -- --topology=2

# 开启运行时包解析（输出到 nms-framework-log.txt）
./ns3 run heterogeneous-nms-main -- --parsePackets=1

# 开启 pcap 抓包
./ns3 run heterogeneous-nms-main -- --pcap=1

# 开启控制/数据双通道（8888/9999）
./ns3 run heterogeneous-nms-main -- --dualChannel=1

# 组合示例：星状拓扑 + 双通道 + 包解析 + pcap，仿真 60s
./ns3 run heterogeneous-nms-main -- --simTime=60 --topology=1 --dualChannel=1 --parsePackets=1 --pcap=1
```

### 仿真后：画图与离线解析

```bash
# 根据 flowmon-heterogeneous.xml 画吞吐/时延/丢包率（PNG + 可选 PDF）
python3 plot-flowmon.py flowmon-heterogeneous.xml -o flowmon-summary --pdf

# 离线解析 pcap（需先安装 scapy，且 pcap 需先由仿真生成）
# 生成 pcap：在 ns-3-dev 根目录运行
./ns3 run heterogeneous-nms-main -- --pcap=1
# 仿真结束后再解析（实际文件名为 heterogeneous-nms-adhoc-<节点ID>-<设备ID>.pcap，以仿真生成为准）：
python3 parse_nms_pcap.py heterogeneous-nms-adhoc-10-1.pcap
# 或列出再选一个： ls heterogeneous-nms-adhoc-*.pcap
```

## 1. 丢包率可视化与性能

### plot-flowmon.py

- 从 FlowMonitor XML 解析 **lostPackets**，并计算 `loss_ratio`、`loss_pct`。
- 新增第 3 个子图：**每条流的丢包率（%）**。
- 支持 `-o/--output` 指定输出前缀、`--pdf` 同时输出 PDF。

**用法示例：**
```bash
python3 plot-flowmon.py flowmon-heterogeneous.xml -o flowmon-summary --pdf
```

### 性能与统计说明

- 丢包率公式与 `flowmon-heterogeneous-loss.txt` 一致：`LostPkts/TxPkts`；FlowMonitor 的 `CheckForLostPackets()` 与 XML 中 `lostPackets` 已用于统计。
- 未改动 TLV 构建/解析与 Socket 逻辑，仅增加可选包解析日志；双通道通过端口区分流，FlowMonitor 按五元组自动分统计。

## 2. WiFi Ad-hoc 自定义拓扑

### 实现位置

`scratch/heterogeneous-nms/framework/heterogeneous-nms-framework.cc`：

- 新增枚举 `HeterogeneousNmsFramework::AdhocTopologyType`：
  - `ADHOC_TOPOLOGY_MESH` (0)
  - `ADHOC_TOPOLOGY_STAR` (1)
  - `ADHOC_TOPOLOGY_CHAIN` (2)
- **ConfigureAdhocMobility()** 按 `m_adhocTopology` 分支：
  - **MESH**：原网格 + RandomWalk2d。
  - **STAR**：节点 0 在中心，其余在半径 40m 圆周上均匀分布，ConstantPosition。
  - **CHAIN**：沿 X 轴等间距 30m 直线排列，ConstantPosition。
- 拓扑与 OLSR、IP 分配解耦，仅改位置与移动模型。

### 使用方式

```bash
./ns3 run heterogeneous-nms-main -- --topology=0   # MESH（默认）
./ns3 run heterogeneous-nms-main -- --topology=1   # STAR
./ns3 run heterogeneous-nms-main -- --topology=2   # CHAIN
```

## 3. 数据包/控制包内容解析

### 运行时解析（NmsPacketParse）

- 当 `NmsPacketParse::SetEnable(true)` 时，在收发路径打日志：HEX + TLV 解析（Type/Len/Value 注释：能量、链路质量、策略内容等）。
- 接入点：`SendPacket`（TLV 发送）、`RecvFromSubnet`（TLV 接收）、`HandleHello`（Hello）、`RecvPolicy` / `HandlePolicyFromSpn`（策略包）。

**命令行开启：**
```bash
./ns3 run heterogeneous-nms-main -- --parsePackets=1
```

- 日志会写入 `nms-framework-log.txt`，并包含 `PKT_PARSE` / `PKT_PARSE_HELLO` / `PKT_PARSE_POLICY` 等事件。

### Pcap 抓包

```bash
./ns3 run heterogeneous-nms-main -- --pcap=1
```

- 生成：`heterogeneous-nms-adhoc-*.pcap`、`heterogeneous-nms-datalink-*.pcap`、`heterogeneous-nms-p2p-*.pcap`。

### 离线解析脚本 parse_nms_pcap.py

- **依赖：** `pip install scapy`
- **用法：** `python3 parse_nms_pcap.py <pcap文件>`
- 解析 UDP payload：TLV(0x01~0x04)、Hello(8B double)、聚合包(4B IP+TLV)，输出 HEX 与明文注释（能量、linkQ、mobility、策略等）。

## 4. 控制/数据双通道分离

### 评估结论（已写在源码注释中）

- **推荐实现双通道**：控制(8888) 与数据(9999) 分离可带来：
  - FlowMonitor 按五元组区分控制流与数据流，便于分别看丢包率、吞吐、时延；
  - 控制通道可单独配置更短间隔/更高优先级；
  - 数据通道可独立做压缩/聚合。
- 若不想改端口，可选用**替代方案**：在发控制包时设置 IP TOS/DSCP 提升优先级。

### 实现要点

- **数据通道**：GMC 监听 9999，子网上报/聚合使用 9999（原 5000/5001 合并为 9999）。
- **控制通道**：Hello / Policy 使用 8888（原 6000/5002/5003 合并为 8888）。
- `HeterogeneousNodeApp::SetChannelPorts(reportPort, spnPolicyPort, nodePolicyPort, helloPort)` 在 `InstallApplications` 中按 `m_useDualChannel` 传入 9999/8888 或原 5001/5002/5003/6000。
- FlowMonitor 已 `InstallAll()`，按 (srcIP, dstIP, protocol, srcPort, dstPort) 自动区分控制流与数据流，无需额外配置。

**使用方式：**
```bash
./ns3 run heterogeneous-nms-main -- --dualChannel=1
```

- 仿真后查看 `flowmon-heterogeneous-loss.txt` 或 XML，按端口 8888/9999 区分控制与数据流统计。

## 5. 命令行参数汇总

| 参数 | 说明 | 默认 |
|------|------|------|
| `--simTime` | 仿真时长(秒) | 100 |
| `--topology` | Adhoc 拓扑 0=MESH, 1=STAR, 2=CHAIN | 0 |
| `--parsePackets` | 运行时包解析日志 | false |
| `--pcap` | 开启 pcap 抓包 | false |
| `--dualChannel` | 控制(8888)/数据(9999) 双通道 | false |
