# 异构 NMS 仿真可视化 — 平台设计与部署说明

本文档合并了**技术选型与架构设计**、**部署运行指南**与**前端使用说明**，便于在 `docs/heterogeneous-nms/` 统一查阅。

---

## 一、技术选型建议

| 方案 | 技术栈 | 与 NS-3 联动 | 优点 | 缺点 | 推荐度 |
|------|--------|--------------|------|------|--------|
| **A. Web 前端** | HTML5 + JavaScript + ECharts + Canvas/SVG | 离线：读 JSON/CSV/日志；实时：WebSocket + 桥接进程 | 无需重编 NS-3、跨平台、易部署、与现有 NetAnim XML/日志兼容 | 需浏览器；实时需额外桥接 | ★★★★★ |
| **B. Python 桌面** | Python 3 + PySide6 + pyqtgraph/NetworkX + Matplotlib | 离线：读文件；实时：Socket 直连或 tail 日志 | 与 NS-3 无编译耦合、开发快、可嵌 C++ 扩展 | 依赖 Python 环境、打包略重 | ★★★★ |
| **C. Qt/C++** | Qt5/6 + QGraphicsView + QChart | 实时：NS-3 内嵌或同进程 Socket | 与 NS-3 同语言、性能最好、可集成进 ns-3 | 开发周期长、需维护 C++/Qt 构建 | ★★★ |

**推荐：以 Web 方案为主**。理由：零侵入 NS-3 可执行文件；双模式（离线读文件 + 可选实时桥接）；界面迭代快；ECharts 做统计图，Canvas/SVG 做拓扑，工业风配色易实现。

## 二、整体架构与数据格式

### 2.1 模块划分

- **拓扑模块**：节点/链路、拖拽缩放、子网着色、时序入网与链路去重。
- **监控模块**：节点状态、筛选器、节点详情面板。
- **回放模块**：时间轴、事件列表、播放/暂停/重置。
- **统计图表模块**：吞吐/时延/丢包、能量/Score 趋势、SPN 选举、导出 PNG/CSV。

数据源：离线为 `topology.json`、`events.json`、`stats.json`（由 `export_visualization_data.py` 从仿真结果目录生成）；实时可选 WebSocket + 桥接进程。

### 2.2 数据格式约定

**topology.json**：`nodes`（id、label、role、subnet、x、y、joinTime、positions、ip、energy、link_quality）、`links`（from、to、type、t）。

**events.json**：带时间戳的事件列表，如 `{ "t": 1.234, "nodeId": 0, "event": "NODE_ONLINE", "details": "..." }`。

**stats.json**：FlowMonitor 流统计、nodeStatesByTime、businessFlows、timeSeries、spnElection* 等。

## 三、依赖与环境

- **前端**：Chrome / Firefox / Edge（ES5+、ECharts）；本地 HTTP 服务：`python3 -m http.server 8000`（或 8080 等）。
- **导出脚本**：Python 3.6+，标准库 json、os、re、xml.etree.ElementTree、argparse。
- **NS-3**：与当前仓库一致版本；编译 `./ns3 build scratch/heterogeneous-nms-framework`；若使用时序入网与 JSONL，需 NS-3 侧实现见 [05-时序入网与JSONL片段.md](05-时序入网与JSONL片段.md)。

## 四、仿真 + 可视化联动步骤（离线，推荐）

1. **运行 NS-3 仿真**  
   得到 `simulation_results/<时间戳>/`，内含 `log/`、`performance/`、`visualization/`（若启用 NetAnim）。

2. **导出可视化数据**（在项目根目录执行）  
   ```bash
   python3 export_visualization_data.py simulation_results/<时间戳> -o visualization/data
   ```  
   生成 `visualization/data/topology.json`、`events.json`、`stats.json`（及可选 meta.json）。若仿真输出 JSONL，导出脚本会解析并写入 nodeStatesByTime 与节点 ip/energy/link_quality。

3. **启动本地 HTTP 服务并打开界面**  
   ```bash
   cd visualization
   python3 -m http.server 8000
   ```  
   浏览器访问 `http://localhost:8000/`。加载：多选 `data/topology.json`、`data/events.json`、`data/stats.json`；或路径输入 `data` 后点击「加载」。

4. **操作**  
   - 子网/类型筛选：勾选 GMC、LTE、Ad-hoc、战术数据链、SPN、TSN；拓扑仅显示选中类型。  
   - 时间轴：底部拖拽或播放/暂停/重置；事件列表点击定位。  
   - 节点详情：点击拓扑节点弹出悬浮信息面板（Node ID、IP、类型、子网、入网状态、能量、链路质量、速度、Score、承载业务、发包/收包）；点击空白关闭。  
   - 业务流筛选：若 stats 含 businessFlows，可勾选要显示的业务流，拓扑上高亮路径。  
   - 统计图表：趋势/性能/SPN 选举，可导出 PNG/CSV。

## 五、界面效果与功能概览

- **拓扑区**：节点按类型着色（GMC 红、SPN 橙、TSN 紫、LTE 蓝、Ad-hoc 绿、数据链黄、未入网灰）；回程红粗实线、数据蓝细虚线；未到 joinTime 灰色，仅两端入网后显示连线；图例标注。
- **下方内容区**：左侧筛选、事件列表、业务流勾选、节点详情占位；右侧统计图表（坐标轴 ≥12px、网格线、dataZoom、导出）。
- **底部**：仿真时间轴、当前时间、播放/暂停/重置。

## 六、常见问题排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 拓扑为空 | 未加载 topology.json 或 nodes 为空 | 先运行导出脚本，再加载 data 下三个 JSON；确认仿真结果目录含 log 与（可选）visualization XML。 |
| 无节点详情 | topology 中无 ip/energy 等，且无 nodeStatesByTime | 若使用 JSONL，确保 NS-3 输出 JSONL 且导出脚本能读到；检查 stats.json 是否含 nodeStatesByTime。 |
| 路径加载失败 | 直接打开 file:// 或路径错误 | 必须通过「加载」选文件，或使用 `python3 -m http.server` 后在同源下用路径 `data` 加载。 |
| 图表不显示 | ECharts 未加载或容器高度为 0 | 确认网络可访问 ECharts CDN；检查 `.chart-box.active` 容器有高度。 |
| Adhoc/TSN 无网状连线 | 导出脚本未补 mesh | 导出脚本会对 Adhoc/TSN 子网补全两两链路；确认 export_visualization_data.py 已更新并重新导出。 |
| 业务流不显示 | 无 ROUTE_START/ROUTE_END 事件 | NS-3 需在业务触发时输出 ROUTE_START details 含 flowId、type、path；见 [05-时序入网与JSONL片段.md](05-时序入网与JSONL片段.md)。 |

## 七、文件清单

- `visualization/index.html` — 主页面  
- `visualization/styles.css` — 样式（工业风、TSN 紫色、布局）  
- `visualization/app.js` — 拓扑、回放、筛选、图表、导出、nodeStatesByTime 支持  
- `export_visualization_data.py` — 导出 topology/events/stats，支持 JSONL、mesh 链路、nodeStatesByTime  
- 数据建议放在 `visualization/data/`，便于路径加载。

---

## 八、可选：实时模式

若需仿真运行过程中实时看到事件与拓扑更新：NS-3 将状态以 JSONL 或 Socket 推送至本机某端口；本地桥接脚本（Python/Node）读取端口或 tail JSONL，通过 WebSocket 推给浏览器；前端连接 WebSocket 后追加事件/状态并刷新。当前仓库默认提供离线流程；实时模式需自行实现桥接与前端 WebSocket 逻辑。
