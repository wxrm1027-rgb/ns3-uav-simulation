# 节点时序入网与场景想定 — 功能验证说明

## 一、joinconfig 节点时序入网

### 1.1 测试步骤

1. **准备 joinconfig**  
   复制 `joinconfig-example.json` 为 `node_join_config.json`，按需修改 `node_id`（须与当前拓扑一致）、`join_time`、`init_pos`、`initial_energy`、`initial_link_quality`。

2. **运行仿真**  
   ```bash
   ./ns3 build heterogeneous-nms-main
   ./ns3 run heterogeneous-nms-main -- --joinConfig=node_join_config.json --simTime=30
   ```

3. **检查日志**  
   - `simulation_results/<时间戳>/log/nms-framework-log_*.txt` 中应有：  
     - `Loaded join config: N nodes from ...`  
     - 若校验失败：`JOINCONFIG_VALIDATION` 及原因（重复 IP、时间戳乱序、数值越界）。  
   - 各节点 `NODE_ONLINE` 时间应与 joinconfig 中 `join_time` 一致（或首次发送在 join_time 时刻）。

4. **检查 JSONL**  
   - `log/nms-state_*.jsonl` 中在 `timestamp < join_time` 时，对应 `node_id` 的 `join_state` 应为 `"not_joined"`，且可带 `initial_energy`、`initial_link_quality`（若配置）；入网后为 `"joined"`。

5. **检查可视化**  
   - 运行 `python3 export_visualization_data.py simulation_results/<时间戳> -o visualization/data`，用本地 HTTP 打开 `visualization/index.html`。  
   - **预期**：拓扑图在 t=0 时无节点（或仅 join_time=0 的节点）；拖动时间轴到某节点 `join_time` 时，该节点在对应坐标出现；节点颜色/状态随事件与 nodeStatesByTime 更新。

### 1.2 预期仿真效果

- 节点在 `join_time` 秒时启动 `HeterogeneousNodeApp`，并写入一条 `NODE_ONLINE`。  
- 若配置了 `init_pos`，节点在该时刻位置被设为给定 (x,y,z)；若配置了 `speed>0` 且为 ConstantVelocity，速度被设置。  
- 若配置了 `initial_energy` / `initial_link_quality`，应用层 UV-MIB 使用该初值，并在 JSONL 中体现。

### 1.3 可视化效果

- 初始：画布仅坐标系/背景，无节点（当所有节点 join_time > 0 时）。  
- 时间轴推进到节点入网时刻：该节点在拓扑图对应坐标出现，并带角色/子网着色。  
- 时间轴回退到入网前：节点消失。  
- 链路仅在两端均已入网后显示。  
- **退网/没电**：当节点发生 NODE_OFFLINE 事件或 JSONL 中能量首次 ≤ 阈值时，视为退网/没电；该时刻之后节点显示为**灰色**，与之相连的链路**不绘制**（断链）。图例中灰色表示「未入网/退网/没电」。

## 二、场景想定模块（当前为骨架）

### 2.1 测试步骤

1. **准备场景配置**  
   使用 `scenario-config-example.json`，或新建 JSON 含 `"scenario_id": "xxx"`。

2. **运行**  
   ```bash
   ./ns3 run heterogeneous-nms-main -- --scenario=scenario-config-example.json --simTime=20
   ```

3. **检查日志**  
   - 应出现 `SCENARIO` 事件：`Loaded scenario: id=test_scenario_01 from ...`。  
   - 当前仅解析 `scenario_id`，轨迹与事件规则为占位，后续可扩展。

### 2.2 扩展说明

- `ScenarioConfig` 含 `trajectoryByNode`、`eventRules`，便于后续接入轨迹驱动与异常事件。  
- 场景配置与 joinconfig 解耦，可通过 `scenario_id` 或同一批 node_id 关联不同场景参数。

## 三、前端时序联动

- **时间轴**：拖动或播放/暂停，`state.currentTime` 更新后重绘拓扑。  
- **节点可见性**：仅当 `currentTime >= node.joinTime` 时参与 `nodeById` 与绘制，实现“到入网时刻才渲染”。  
- **链路**：仍按 `link.t` 与两端 `joinTime` 判断，仅两端入网后显示。  
- **导出**：`export_visualization_data.py` 从事件与 JSONL 得到 `joinTime`、positions，写入 `topology.json` 与 `stats.nodeStatesByTime`，供前端一致展示。
- **未使用 joinconfig 时**：若仿真未配置入网（所有 NODE_ONLINE 均在 0s），导出脚本会检测“所有节点 joinTime 相同”，并自动按节点 ID 错峰（0, 0.5, 1.0, … 秒），使打开页面时仍能呈现“先少后多”的入网过程；使用 joinconfig 或 JSONL 中有真实入网时间时不会覆盖，仍按真实时间导出。

---

## 四、问题定位

- **配置解析错误**：查看 `JOINCONFIG_VALIDATION`、`SYSTEM` 中 “Loaded join config” 的节点数。  
- **节点未出现**：确认 `node_id` 与仿真拓扑一致；确认 `join_time` 单位（秒/毫秒）与时间轴范围。  
- **可视化无节点**：确认已导出 data 并加载 `topology.json`、`events.json`、`stats.json`；确认 `joinTime` 在 topology 的 nodes 中正确写入。  
- **退网/变灰/断链**：`offlineTime` 由导出脚本从 NODE_OFFLINE 事件与 JSONL 中首次能量 ≤ 0.01 的时刻取较早值写入 `topology.json`；若仿真未产生 NODE_OFFLINE 或 JSONL 中无能量耗尽，则不会出现退网灰与断链。
