# 异构 NMS 框架全量优化：使用说明与验证步骤

本文档对应 `scratch/heterogeneous-nms/framework/heterogeneous-nms-framework.cc` 的优化说明：自组网拓扑重构、仿真文件结构化归档、抓包解析与自动输出、通用约束与注释规范。（NetAnim 已下线）

---

## 一、自组网拓扑配置（仅 WiFi Ad-hoc 子网）

### 1.1 拓扑类型

| 类型 | 枚举/值 | 说明 |
|------|---------|------|
| 网状 Mesh | `0` / `mesh` | 全联通，所有节点两两直连；位置为网格+RandomWalk |
| 星型 Star | `1` / `star` | 指定 1 个中心节点（容器中 index 0），其余仅与中心直连；位置为中心+圆周 |
| 树型 Tree | `2` / `tree` | 按节点 ID 层级：根(0)、一级子(1,2)、二级子(3,4)；位置按层级摆放 |

### 1.2 命令行配置（无需改代码）

```bash
# 方式一：数字
./ns3 run heterogeneous-nms-main -- --topology=0    # MESH
./ns3 run heterogeneous-nms-main -- --topology=1    # STAR
./ns3 run heterogeneous-nms-main -- --topology=2    # TREE

# 方式二：字符串（推荐）
./ns3 run heterogeneous-nms-main -- --adhoc-topology=mesh
./ns3 run heterogeneous-nms-main -- --adhoc-topology=star
./ns3 run heterogeneous-nms-main -- --adhoc-topology=tree
```

### 1.3 验证拓扑与链路

- 运行后使用日志与导出 JSON 检查拓扑与链路是否正确。
- **Star**：应看到仅“叶子”节点连到中心，连线描述为 `Adhoc-Star`。
- **Tree**：应看到父子层级连线，描述为 `Adhoc-Tree`。
- **Mesh**：应看到任意两节点间有一条连线，描述为 `Adhoc-Mesh`。

## 二、可视化说明（NetAnim 已下线）

### 2.1 图标尺寸与配色

- **GMC（Node 0）**：尺寸 15×15，深红色 RGB(200,0,0)。
- **SPN 节点**（LTE-eNB、Adhoc-SPN、DL-SPN）：尺寸 12×12，橙色 RGB(255,165,0)。
- **普通业务节点**：尺寸 10×10；LTE-UE 蓝色、Adhoc-Node 绿色、DL-Node 黄色。
- **EPC 节点（1/2/3）**：保持隐藏，不设置尺寸与颜色，避免越界。

### 2.2 节点显示顺序说明

- 业务节点按 **GMC(0) → LTE-eNB → LTE-UE → Adhoc → DataLink** 的逻辑顺序在代码中依次调用 `UpdateNodeDescription/Color/Size`。
- NetAnim 的「Node List」按 **节点 ID 升序** 显示；EPC 节点(1,2,3) 由 ns-3 创建顺序决定，仍会出现在列表中但已隐藏，不参与渲染。若需 EPC 排在列表最后，需调整节点创建顺序（涉及 LTE/EPC 内部结构，未在本优化中修改）。

### 2.3 避免段错误

- 所有 `UpdateNodeSize` / `UpdateNodeColor` 仅对非空 `Ptr<Node>` 调用，且不对已隐藏的 EPC 节点(1,2,3) 调用，避免 NetAnim 越界。

## 三、仿真文件结构化归档

### 3.1 四级目录结构

- **第 1 级**：根目录 `simulation_results/`（不存在则自动创建）。
- **第 2 级**：时间戳子目录 `simulation_results/YYYY-MM-DD_HH-MM-SS`。
- **第 3 级**：时间戳目录下按类型拆分的 4 个子目录：
  - `visualization/`：可视化预留目录；
  - `performance/`：性能统计（flowmon txt/xml、可选 png 图表）；
  - `log/`：仿真日志；
  - `packet/`：pcap 抓包原始文件 + 抓包解析文本。

### 3.2 输出文件路径（均带时间戳，避免覆盖）

| 类型 | 路径示例 |
|------|----------|
| 可视化 | `simulation_results/${时间戳}/visualization/heterogeneous-nms-framework_${时间戳}.xml` |
| 性能统计 | `simulation_results/${时间戳}/performance/flowmon_stats_${时间戳}.txt`，`.../flowmon_stats_${时间戳}.xml`，可选 `.../performance_charts_${时间戳}.png` |
| 仿真日志 | `simulation_results/${时间戳}/log/nms-framework-log_${时间戳}.txt` |
| 抓包 | `simulation_results/${时间戳}/packet/packet_capture_${时间戳}_adhoc-*.pcap` 等 |
| 抓包解析 | `simulation_results/${时间戳}/packet/packet_parse_result_${时间戳}.txt` |

### 3.3 工具函数

- `GetOutputDir()`：返回当前时间戳目录（第 2 级）。
- `GetTimestamp()`：返回时间戳字符串。
- `MakeOutputPath(baseName, ext)`：兼容接口，生成 `performance/` 下路径。
- `MakeOutputPathInCategory(category, baseName, ext)`：按分类生成路径，`category` 取 `"visualization"`、`"performance"`、`"log"`、`"packet"`。
- `GetPacketDir()`：返回 `packet/` 子目录路径（供 pcap 前缀与解析脚本使用）。
- 多级目录在仿真启动时统一创建，路径拼接使用 `m_outputDir + "/" + category + "/" + baseName + "_" + m_timestamp + ext`，兼容 Linux 权限与路径规范。

## 四、抓包与自动解析

### 4.1 抓包

- 使用 `--pcap=1` 时，为 Adhoc、DataLink、P2P 回程开启 pcap，所有 pcap 文件写入归档目录，前缀 `packet_capture_<时间戳>_adhoc` / `_datalink` / `_p2p`。

### 4.2 自动解析

- 仿真结束后，若已开启 pcap，框架会自动调用：
  `python3 parse_nms_pcap.py <归档目录> -o packet_parse_result_<时间戳>.txt`
- 解析脚本会遍历该目录下所有 `*.pcap`，按时间戳排序，输出格式：
  **时间戳 | 包类型 | 源节点 | 目的节点 | 核心字段 | 备注**
- 包类型区分：TLV(业务)、Hello(选举)、Aggregate(SPN 聚合)、Policy 等；核心字段含 energy/linkQ/mobility、Score、deltaEnergy、策略内容等。

### 4.3 依赖与手动解析

- 自动解析依赖系统已安装 `python3` 与 `scapy`（如 `sudo apt install python3-scapy`）。
- 若未安装 scapy，自动解析会静默失败，不影响仿真与 pcap 生成；可事后手动执行：
  `python3 parse_nms_pcap.py simulation_results/2026-03-08_15-30-45 -o simulation_results/2026-03-08_15-30-45/packet_parse_result_2026-03-08_15-30-45.txt`

## 五、效果验证步骤（终端命令示例）

### 5.1 编译

```bash
cd /path/to/ns-3-dev
./ns3 build heterogeneous-nms-main
```

### 5.2 运行（归档 + 星型拓扑 + pcap + 短时仿真）

```bash
./ns3 run heterogeneous-nms-main -- \
  --simTime=20 \
  --adhoc-topology=star \
  --pcap=1
```

### 5.3 检查归档与结果

```bash
# 查看本次仿真归档目录
ls simulation_results/

# 进入最新一次时间戳目录（替换为实际名称）
cd simulation_results/2026-03-08_15-30-45

# 检查文件
ls -la
# 应包含：flowmon_stats_*.xml, flowmon_stats_*.txt,
#         nms-framework-log_*.txt, packet_capture_*_adhoc-*.pcap 等,
#         以及 packet_parse_result_*.txt（若已安装 scapy）

# 查看性能统计
head -20 flowmon_stats_*.txt

# 查看解析结果（若存在）
head -30 packet_parse_result_*.txt
```

### 5.4 验证可视化数据

- 运行 `export_visualization_data.py` 后检查 `visualization/data/*.json` 内容。
- 检查：GMC 大图标深红、SPN 橙色中图标、普通节点小图标且按子网着色；Adhoc 为星型连线；无崩溃或段错误。

### 5.5 生成性能图表（可选）

- 使用 `plot-flowmon.py` 对归档内 flowmon XML 画图并保存到同目录：

```bash
python3 plot-flowmon.py simulation_results/2026-03-08_15-30-45/flowmon_stats_2026-03-08_15-30-45.xml \
  -o simulation_results/2026-03-08_15-30-45/performance_charts_2026-03-08_15-30-45 \
  --pdf
```

## 六、核心代码修改位置速查

| 需求 | 文件 | 位置/函数 |
|------|------|-----------|
| 拓扑 Star/Tree/Mesh | heterogeneous-nms-framework.cc | `AdhocTopologyType` 枚举、`ConfigureAdhocMobility()`、`SetupMonitoring()` 中 Adhoc 连线 switch |
| 归档与时间戳 | 同上 | `Run()` 开头创建目录、`MakeOutputPath()`、所有输出使用 `MakeOutputPath` 或 `m_outputDir` |
| NetAnim 尺寸/配色 | 同上 | `SetupMonitoring()` 中 `UpdateNodeSize`/`UpdateNodeColor` 统一尺寸与 RGB |
| pcap 归档 | 同上 | `BuildAdhocSubnet`/`BuildDataLinkSubnet`/`SetupBackhaul` 中 `EnablePcap(prefix)` 使用 `m_outputDir + "/packet_capture_" + m_timestamp + "_..."` |
| 自动解析 | 同上 | `Run()` 末尾 `system("python3 parse_nms_pcap.py ...")` |
| 解析脚本目录+格式 | parse_nms_pcap.py | 支持目录输入、`-o` 输出、行格式「时间戳 \| 包类型 \| 源节点 \| 目的节点 \| 核心字段 \| 备注」 |
| 命令行 topology | 同上 | `main()` 中 `--topology`、`--adhoc-topology=star|tree|mesh` |
